# Software Engineer — CV Expectations & Hiring Criteria

## What Recruiters Look For (First 6 Seconds)
- Clear job titles that match the role applied for
- Quantified technical achievements (latency reductions, scale, users, uptime)
- Recognisable company names, tech stack, or projects
- Evidence of owning features end-to-end, not just contributing

## Must-Have CV Sections
- **Experience** — with measurable bullet points in Action + Result format
- **Technical Skills** — clearly listed languages, frameworks, tools, cloud platforms
- **Projects** — especially for junior/mid engineers; links to GitHub or live demos add credibility
- **Education** — degree and institution; relevant modules optional unless junior

## Common Weaknesses in SWE CVs
- Bullets describe responsibilities, not achievements ("Worked on backend" vs "Reduced API latency by 40%")
- No metrics — scale, speed, reliability numbers are expected for mid/senior roles
- Generic skill lists with no evidence of depth ("Python, JavaScript, SQL" with no context)
- Listing every technology ever touched rather than the ones used consistently
- Missing keywords for the specific stack the company uses

## Strong Bullet Examples
- "Architected a distributed caching layer using Redis that reduced database load by 60% and improved P99 latency from 800ms to 120ms"
- "Led migration of monolithic Django app to microservices, cutting deployment time from 45 minutes to 8 minutes"
- "Built CI/CD pipeline using GitHub Actions and Docker that reduced manual release steps from 12 to 2"

## Keywords Recruiters and ATS Look For
Languages: Python, JavaScript, TypeScript, Java, Go, Rust, C++, C#
Frameworks: React, Node.js, FastAPI, Django, Flask, Spring Boot, Next.js
Infrastructure: Docker, Kubernetes, AWS, GCP, Azure, Terraform, CI/CD
Data: PostgreSQL, MySQL, MongoDB, Redis, Kafka, Elasticsearch
Practices: Agile, Scrum, TDD, code review, system design, REST API, GraphQL, microservices

## Seniority Signals
- **Junior**: academic projects, internships, personal projects, bootcamp
- **Mid**: owns features, improves existing systems, mentors juniors
- **Senior**: designs systems, drives architectural decisions, measurable business impact
- **Staff/Principal**: cross-team technical leadership, long-term platform strategy

## Red Flags
- Bullet points starting with "Responsible for" or "Helped with"
- No version control mentioned (Git is expected at all levels)
- Gaps with no explanation in a competitive market
- CV longer than 2 pages for under 8 years experience
